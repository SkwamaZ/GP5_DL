# Data dictionary

Два датасета связаны по ключу **`nmId`** (артикул WB). Отзывы берутся только по тем
`nmId`, что есть в спарсенном каталоге.

## 1. Каталог товаров — `data/interim/catalog.parquet`

Источник: публичная поисковая выдача WB (`search.wb.ru`, `resultset=catalog`).
Одна строка — одна карточка товара.

| поле | тип | значения / смысл |
|---|---|---|
| `nmId` | int64 | артикул WB; ключ связи с отзывами; уникален в датасете |
| `name` | string | название карточки |
| `brand` | string | бренд (может быть пустым) |
| `category` | string | наша категория: `glasses` / `phone_cases` / `hangers` |
| `subject_id` | int | id предметной категории WB (служебный) |
| `price` | float | доступная цена, ₽. Берётся из `sizes[].price` (`total` / `product` / `basic`, делим на 100). Это та цена, что отдаёт ручка; итоговую с СПП не гарантируем |
| `rating` | float | средний рейтинг карточки, 0–5 (`reviewRating`). **Таргет табличной задачи.** `0.0` = у карточки нет отзывов |
| `n_reviews` | int | число отзывов на карточке (`feedbacks`) |
| `n_photos` | int | число фото в карточке (`pics`) |
| `colors` | string | цвета через `; ` (может быть пустым) |
| `n_colors` | int | число цветов |
| `supplier` | string | продавец (может быть пустым) |

## 2. Отзывы — `data/interim/reviews.parquet` (+ `reviews.jsonl`)

Источник: локальный CC0-датасет `wb-feedbacks` (jsonl в zstd), отфильтрованный по `nmId`
каталога и стратифицированный по оценке. Одна строка — один отзыв.

| поле | тип | значения / смысл |
|---|---|---|
| `nmId` | int64 | артикул WB; ключ связи с каталогом; только существующие (спарсенные) товары |
| `category` | string | категория товара (подтянута из каталога по `nmId`) |
| `mark` | int | оценка отзыва, 1–5 (`productValuation`). **Таргет текстовой задачи** |
| `color` | string | цвет товара, указанный в отзыве (может быть пустым) |
| `text` | string | текст отзыва (может быть пустым) |
| `answer` | string | ответ продавца на отзыв (часто пустой) |

## 3. Чистый табличный датасет — `data/processed/tabular_*.parquet`

`tabular_clean.parquet` — весь чистый датасет, `tabular_train/val/test.parquet` — сплиты
70/10/20 (seed из конфига). Чистка: убраны дубликаты `nmId`, карточки без отзывов
(`rating = 0`, `n_reviews = 0` — нет таргета) и с нулевой ценой; пустой бренд заменён
на `unknown`. Колонки: `nmId`, `name`, `brand`, `category`, `price`, `rating`,
`n_reviews`, `n_photos`, `n_colors` (см. раздел 1; служебные `subject_id`, `colors`,
`supplier` отброшены) плюс:

| поле | тип | значения / смысл |
|---|---|---|
| `title_len` | int | длина названия карточки в символах (новый признак) |

Кодирование и скейлинг в файлы не зашиты — они выполняются пайплайном
`src/preprocess.py::build_tabular_preprocessor` (лог-скейлинг `price`/`n_reviews`,
StandardScaler, one-hot для `category`, target-encoding для `brand`), который фитится
**только на train**.

## 4. Чистый текстовый датасет — `data/processed/text_*.parquet`

`text_clean.parquet` — весь чистый датасет, `text_train/val/test.parquet` — сплиты 70/10/20,
**стратифицированные по `mark`**. Чистка: убраны полные дубли (`nmId`+`text`+`mark`) и отзывы,
пустые после нормализации. Поля отзывов (см. выше) плюс:

| поле | тип | значения / смысл |
|---|---|---|
| `text_norm` | string | нормализованный текст: нижний регистр, `ё`→`е`, без ссылок, эмодзи и мусорных символов, схлопнуты повторы знаков и пробелы |
| `n_words` | int | длина `text_norm` в словах |
| `aspect_defect` | bool | в тексте упомянут брак/поломка (правила в `src/aspects.py`: слова отзыва приводятся к леммам через pymorphy3 и сверяются со словарями, плюс устойчивые фразы) |
| `aspect_size` | bool | упомянут размер/посадка |
| `aspect_delivery` | bool | упомянута доставка/упаковка |
| `aspect_quality` | bool | упомянуто качество/материалы |
| `aspect_price` | bool | упомянута цена/стоимость |

Веса классов для дисбаланса (посчитаны по train) — `data/processed/class_weights.json`.

## Снапшот и баланс

Объёмы, распределение оценок 1–5 и разбивка по категориям фиксируются в
`docs/snapshot.md` / `docs/snapshot.json` (генерируются `src/make_snapshot.py`),
графики — в ноутбуке EDA (`notebooks/eda.ipynb`).
