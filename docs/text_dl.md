# Текстовая задача: классификация оценки отзыва 1–5 (этап 5)

По тексту отзыва предсказываем оценку от 1 до 5, основная метрика — macro-F1, потому что классы сильно несбалансированы (пятёрок втрое больше всех остальных вместе). Сплиты те же, что с этапа 3 (23229 / 3319 / 6638), seed 42, всё обучение на mps. Сравниваем четыре значимо разные архитектуры на одних данных: TF-IDF с логрегом как точку отсчёта без сети, свёрточную сеть по словам, двунаправленный LSTM и дообучение трансформера ruBERT-tiny2 через высокоуровневый Trainer HuggingFace. Каждый запуск — отдельный run в DagsHub (эксперимент wb-review-sentiment) с гиперпараметрами, метриками по эпохам, кривыми обучения и весами модели

Весь код этапа — в notebooks/text_dl.ipynb

## Данные и дисбаланс

Распределение оценок в train: 1 — 2909, 2 — 1134, 3 — 2134, 4 — 3310, 5 — 13742. Перекос в сторону пятёрок огромный, поэтому смотрим именно macro-F1, а не accuracy. Дисбаланс лечим по-разному, чтобы показать оба приёма: у TF-IDF — встроенный class_weight balanced, у CNN и LSTM — веса классов с этапа 3 в кросс-энтропии, у трансформера — сбалансированная подвыборка train поровну по классам (1134 на класс, 5670 всего). Подвыборка для трансформера это ещё и осознанное упрощение под M3: на ноутбуке без видеокарты учить tiny-BERT на полном train незачем. Оцениваем все модели на одних и тех же полных val и test, так что сравнение честное

## Четыре архитектуры

| Архитектура     | Суть                                                                       | Борьба с дисбалансом        |
| --------------- | -------------------------------------------------------------------------- | --------------------------- |
| tf-idf + логрег | TF-IDF (30000 признаков, слова и биграммы) + LogisticRegression            | class_weight balanced       |
| cnn-text        | эмбеддинги 128, Conv1d с окнами 3/4/5 по 64 фильтра, max-pool, dropout 0.4 | веса классов в лоссе        |
| bilstm          | эмбеддинги 128, двунаправленный LSTM 128, конкатенация последних состояний | веса классов в лоссе        |
| rubert-tiny2    | дообучение cointegrated/rubert-tiny2 (3 слоя, скрытый 312) через Trainer    | сбалансированная подвыборка |

Подход не из лекций — трансформер: предобученный русский BERT дообучается готовым высокоуровневым Trainer, без ручного цикла. Trainer держит early stopping и возвращает лучшую по val macro-F1 модель, eval считаем и на val, и на сбалансированном train, поэтому train и val macro-F1 по эпохам выходят как у своих сетей. В артефакт DagsHub кладём и модель, и токенизатор

## Обучение и метрики по эпохам

Свои сети: Adam, кросс-энтропия с весами классов, batch 64, до 10 эпох, early stopping по val macro-F1 с patience 3, лучшие веса восстанавливаются, метрики train и val логируются по эпохам. Трансформер: Trainer с per-epoch eval и early stopping, и train, и val macro-F1 пишутся по эпохам. Видно здоровую картину обучения: у победившей конфигурации трансформера val macro-F1 растёт по эпохам с 0.46 до 0.485 и далее до 0.491, train идёт чуть выше val — переобучения нет, кривые обучения сохранены в DagsHub

## Подбор гиперпараметров

Берём сильнейшую из двух своих сетей по val (это BiLSTM) и гоняем сетку: скорость обучения из двух значений как главный параметр оптимизации и dropout из двух — основную регуляризацию для коротких текстов, где сеть быстро переобучается. Для трансформера отдельно пробуем вторую конфигурацию с другой скоростью обучения и более коротким max_len: на коротких отзывах меньше длина — быстрее обучение и меньше паддинга

| Запуск                | val macro-F1 | Лучшая эпоха |
| --------------------- | ------------ | ------------ |
| tfidf_logreg          | 0.5116       | —            |
| bert_tiny_lr5e5_len64 | 0.4907       | 3            |
| bilstm (база)         | 0.4846       | 3            |
| bilstm_lr0.002_do0.5  | 0.4800       | 3            |
| bilstm_lr0.0005_do0.3 | 0.4785       | 4            |
| bert_tiny (база)      | 0.4749       | 3            |
| bilstm_lr0.002_do0.3  | 0.4704       | 4            |
| bilstm_lr0.0005_do0.5 | 0.4682       | 5            |
| cnn                   | 0.4654       | 5            |

Сетка вокруг BiLSTM двигает метрику в пределах сотых — архитектура устойчива, подбор скорее подтверждает базовую конфигурацию, чем переворачивает её. Для трансформера вторая конфигурация (lr 5e-5, max_len 64, 3 эпохи) заметно лучше первой (0.4907 против 0.4749): tiny-модели на коротких текстах нужна скорость обучения побольше

## Финальная оценка на test

Test использован один раз в самом конце, подбор — только по val. Для каждой архитектуры взят её лучший по val вариант:

| Архитектура     | test macro-F1 |
| --------------- | ------------- |
| tf-idf + логрег | 0.5159        |
| rubert-tiny2    | 0.5011        |
| cnn-text        | 0.4685        |
| bilstm          | 0.4591        |

Отчёт по классам у победителя (tf-idf + логрег), macro-F1 0.516, accuracy 0.715:

| Оценка | precision | recall | F1    | отзывов |
| ------ | --------- | ------ | ----- | ------- |
| 1      | 0.597     | 0.626  | 0.611 | 831     |
| 2      | 0.205     | 0.247  | 0.224 | 324     |
| 3      | 0.315     | 0.348  | 0.330 | 610     |
| 4      | 0.479     | 0.545  | 0.510 | 946     |
| 5      | 0.942     | 0.870  | 0.904 | 3927    |

Матрица ошибок у всех моделей одной формы: пятёрки берутся уверенно (F1 0.90), а соседние классы путаются между собой, хуже всего двойки и тройки (F1 0.22 и 0.33) — их мало и текстом они близки к единицам и четвёркам

Честный вывод по архитектурам: на этих данных классика держит первое место, дообученный ruBERT-tiny2 почти догоняет, а свои сети по эмбеддингам отстают. Это закономерно для коротких отзывов: половина не длиннее 8 слов, основной сигнал несут отдельные слова и биграммы вроде «не работает» или «отличные», и линейная модель по TF-IDF снимает его без потерь, тогда как обучаемым с нуля эмбеддингам не хватает примеров по редким классам. Глубина сама по себе тут не выигрывает — выигрывает модель, которой хватает сигнала и данных. ruBERT-tiny2 при этом учится осмысленно и на сбалансированной подвыборке всего в 5670 отзывов уже выходит на уровень классики, так что на полном train и более тяжёлой модели он бы её обошёл, но это уже не про M3 — он остаётся правильным выбором на вырост, когда отзывов станет кратно больше

## Аспекты недовольства

На негативных отзывах (оценка 1–2) считаем долю аспектов по словарю лемм с этапа 3. Негативных — 5776 из 33186, и причины по категориям разные:

| Категория | Негативных | Брак  | Размер | Доставка | Качество | Цена  |
| --------- | ---------- | ----- | ------ | -------- | -------- | ----- |
| очки      | 2212       | 0.258 | 0.087  | 0.081    | 0.127    | 0.140 |
| чехлы     | 1922       | 0.148 | 0.148  | 0.062    | 0.102    | 0.133 |
| вешалки   | 1642       | 0.219 | 0.051  | 0.110    | 0.272    | 0.149 |

Поверх словаря — тематическое моделирование LDA отдельно по каждой категории, чтобы достать живые формулировки жалоб. По очкам темы про стёкла, сломанные дужки, кривые оправы и возврат, по чехлам — не нажимаются кнопки, не подходит под модель, не совпадает вырез под камеру, по вешалкам — хлипкая конструкция, гнётся и падает под весом, ломается сразу, запах пластика

## Бизнес-вывод по трём категориям

Рейтинг роняет сам товар и точность карточки, а не логистика — доставка и цена вторичны во всех трёх категориях. В очках главная боль — брак (25.8%): сломанные дужки, треснувшие стёкла и линзы, поцарапанный товар, лечится контролем упаковки и прочности. В чехлах перекос смещён к посадке: брак и размер вровень (по 14.8%), и темы говорят про несоответствие карточки реальному телефону, а не явный дефект — нужно честное соответствие модели и фото. В вешалках лидируют качество (27.2%) и брак (21.9%): хлипкая конструкция, ломается под весом, запах дешёвого пластика — работать надо с заявленной нагрузкой и материалом.
