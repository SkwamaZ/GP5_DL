# Табличная задача: MLP для среднего рейтинга (этап 4)

Регрессия среднего рейтинга карточки по её атрибутам: цена, число фото, бренд, категория, число отзывов, число цветов, длина названия. Сплиты с этапа 3 (7309 / 1045 / 2089), препроцессор `build_tabular_preprocessor` фитится только на train: log1p + StandardScaler для цены и числа отзывов, one-hot для категории, target encoding с KFold для бренда. Метрики RMSE / MAE, seed 42, обучение на `mps`. Каждый запуск - отдельный run в DagsHub (эксперимент `wb-tabular-rating`) с гиперпараметрами, метриками по эпохам, кривыми обучения и весами модели

Важный контекст из EDA: целевая очень узкая (среднее 4.83, std 0.14), почти весь каталог - карточки 4.5–5.0. Поэтому даже предсказание константой даёт RMSE около 0.15, и борьба идёт за сотые. Ценность модели - не точное среднее, а ранжирование редких проблемных карточек

Весь код этапа - в `notebooks/tabular_mlp.ipynb`.

Оценка на val:

| Среднее по train              | 0.1399   | 0.1065  |

Бустинг - главный соперник сети: сеть имеет смысл, только если обходит 0.1295

## Конфигурации MLP

Adam, MSE-лосс, batch 64, до 50 эпох, early stopping по val RMSE с patience 10, лучшие веса восстанавливаются. Четыре значимо разные конфигурации:

| Конфигурация     | Архитектура                                           | val RMSE | val MAE | Лучшая эпоха |
| ---------------- | ----------------------------------------------------- | -------- | ------- | ------------ |
| mlp_small        | [64], ReLU, без регуляризации                         | 0.1274   | 0.0875  | 49           |
| mlp_deep_dropout | [256, 128, 64], ReLU, dropout 0.2, wd 1e-4            | 0.1537   | 0.1186  | 2            |
| mlp_bn_gelu      | [128, 64], GELU, BatchNorm                            | 0.1432   | 0.1088  | 36           |
| mlp_brand_emb    | [128, 64], ReLU, dropout 0.1, эмбеддинг бренда dim 16 | 0.1718   | 0.1203  | 49           |

Подход не из лекций - mlp_brand_emb: вместо target encoding бренд подаётся в `nn.Embedding` (с `padding_idx=0`, чтобы бренды не из train получали нейтральный нулевой вектор, а не случайный) и сеть учит представление бренда сама, как в рекомендательных системах

Как архитектура влияет на качество:

1. Выиграла самая маленькая сеть. Данных 7.3 тыс. строк и 9 входных признаков - ёмкость глубокой сети здесь не во что вложить, она только добавляет способов переобучиться
2. mlp_deep_dropout лучшую val-метрику показала уже на 2-й эпохе и дальше не улучшалась: dropout 0.2 на такой мелкой задаче слишком грубая регуляризация, сеть не может дотянуться до сотых, за которые идёт борьба
3. BatchNorm + GELU хуже простого ReLU: статистики нормализации по батчу 64 шумные, а узкая целевая чувствительна к этому шуму
4. Эмбеддинг бренда переобучается: 1137 брендов дают ~18 тыс. обучаемых параметров только в эмбеддинге при 7.3 тыс. строк train. Target encoding с KFold-регуляризацией использует ту же информацию экономнее. Идея осталась в проекте как подход не из лекций, но для этого объёма данных она избыточна

## Подбор гиперпараметров

Сетка вокруг лучшей архитектуры mlp_small: lr из {0.0003, 0.001, 0.003} \* weight decay из {0, 0.0001}. Перебирали именно это, потому что lr - главный параметр оптимизации, а weight decay - единственная регуляризация, у которой был шанс помочь маленькой сети

| Запуск               | val RMSE   | val MAE    | Лучшая эпоха |
| -------------------- | ---------- | ---------- | ------------ |
| lr 0.0003, wd 0      | 0.1289     | 0.0909     | 40           |
| lr 0.0003, wd 0.0001 | 0.1291     | 0.0932     | 49           |
| **lr 0.001, wd 0**   | **0.1274** | **0.0875** | 49           |
| lr 0.001, wd 0.0001  | 0.1282     | 0.0892     | 22           |
| lr 0.003, wd 0       | 0.1287     | 0.0884     | 12           |
| lr 0.003, wd 0.0001  | 0.1286     | 0.0883     | 12           |

Разброс в пределах 0.0017 RMSE - архитектура устойчива к гиперпараметрам, сетка скорее подтверждает выбор, чем меняет его. Маленький lr не успевает за 50 эпох, большой быстро сходится, но чуть менее точно. Победитель - исходный lr 0.001 без weight decay.

## Финальная оценка на test

Test использован один раз в самом конце (подбор - только по val):

| Модель                        | test RMSE  | test MAE |
| ----------------------------- | ---------- | -------- |
| Среднее по train              | 0.1511     | 0.1058   |
| HistGradientBoostingRegressor | **0.1418** | 0.0871   |
| MLP [64] (lr 0.001)           | 0.1448     | 0.0900   |

По val сеть была чуть впереди (0.1274 против 0.1295), на test порядок поменялся: бустинг 0.1418, MLP 0.1448. Обе разницы - в третьем знаке, то есть в пределах шума узкой целевой. Честный вывод: на 7 тыс. строк табличных данных MLP корректно обучается и уверенно бьёт константу, но устойчивого преимущества над градиентным бустингом не даёт - для продакшена здесь разумнее бустинг, сеть остаётся демонстрацией DL-подхода и базой для сравнения с текстовой задачей

## Важность признаков

Permutation importance лучшей модели на test: признак перемешивается 5 раз, смотрим средний прирост RMSE. One-hot категории и кодировка бренда перемешиваются как один исходный столбец, поэтому важность считается на уровне признака, а не колонки матрицы

| Признак   | Прирост RMSE |
| --------- | ------------ |
| brand     | 0.0124       |
| n_reviews | 0.0040       |
| category  | 0.0026       |
| price     | 0.0023       |
| n_photos  | 0.0012       |
| title_len | 0.0001       |
| n_colors  | 0.0000       |

Бизнес-инсайт: рейтинг почти целиком определяется тем, кто продаёт. Бренд один даёт больше, чем все остальные признаки вместе, на втором месте - накопленное число отзывов. Косметика карточки (число фото, цветов, длина названия) на средний рейтинг почти не влияет -
улучшать оформление ради рейтинга бессмысленно, работать нужно с товаром и продавцом. Практический фильтр зоны риска: малоизвестный бренд + мало отзывов + подозрительная цена. Почему конкретно карточка падает в рейтинге - отвечает текстовая модель по отзывам: по EDA главные причины - брак, размер и качество материалов
